    Fix 026 & 027 Implementation Plan

    For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

    Goal: Fix double-emission of parse errors on stderr (026) and fix missing continuation prompt for incomplete input (027).

    Architecture: Both fixes are in src/plcc/parser/table_cli.py. Fix 026 removes two verbose.emit_error() calls that wrongly treat parse results as tool diagnostics. Fix 027 adds a break on found=="eof" to stop the retry loop that generates cascade errors, restoring parse.py's only_eof_errors detection.

    Tech Stack: Python 3, pytest

    Task 1: Fix 026 — parse errors must not appear on stderr

    Files: - Modify: src/plcc/parser/table_cli_test.py - Modify: src/plcc/parser/table_cli.py

    • [ ] Step 1: Write the failing test

    Add to src/plcc/parser/table_cli_test.py:

    def test_parse_error_does_not_write_to_stderr(capsys, monkeypatch):
    # Grammar: program → NUM; token: PLUS → parse error.
    # Parse errors are results (stdout JSONL), not diagnostics (stderr).
    ll1_file = tempfile.NamedTemporaryFile(mode="w", suffix=".json", delete=False)
    try:
        json.dump(_TRIVIAL_LL1, ll1_file)
        ll1_file.flush()
        ll1_file.close()
        bad_tokens = [_tok("PLUS", "+"), _sentinel()]
        stdin_data = "\n".join(json.dumps(t) for t in bad_tokens) + "\n"
        monkeypatch.setattr("sys.stdin", io.StringIO(stdin_data))
        try:
            run_main([f"--ll1={ll1_file.name}"])
        except SystemExit:
            pass
        _, err = capsys.readouterr()
        assert err == ""
    finally:
        os.unlink(ll1_file.name)

    • [ ] Step 3: Remove the two verbose.emit_error() calls for parse result errors

    Open src/plcc/parser/table_cli.py. There are four verbose.emit_error() call sites. Keep the two that guard tool-internal failures (lines ~44 and ~48: cannot load ll1.json and grammar is not LL(1)). Remove the other two:

    1. In the consumed == 0 branch (inside the while loop), remove: python verbose.emit_error(tok.get("source", {}), record["message"])

    2. In the except ParseError block, remove: python verbose.emit_error(e.source, str(e))

    After removal the consumed == 0 branch looks like:

    if consumed == 0:
    tok = tokens[cursor]
    record = {
        "kind": "error",
        "message": f"unexpected token {tok.get('name', '?')!r}",
        "stage": "plcc-parser-table",
        "source": tok.get("source", {}),
    }
    print(json.dumps(record), flush=True)
    cursor += 1
    continue

    And the except ParseError block looks like:

    except ParseError as e:
    record = {
        "kind": "error",
        "message": str(e),
        "stage": "plcc-parser-table",
        "source": e.source,
    }
    if e.found:
        record["found"] = e.found
    print(json.dumps(record), flush=True)
    cursor += 1

    Task 2: Fix 027 — break on found=="eof" to prevent cascade errors

    Files: - Modify: src/plcc/parser/table_cli_test.py - Modify: src/plcc/parser/table_cli.py

    • [ ] Step 1: Add the grammar fixture and write the failing test

    Add to src/plcc/parser/table_cli_test.py, after the existing fixtures:

    # Grammar: exp → NUM exp2 ; exp2 → OP exp | ε
# Demonstrates incomplete input: "23 +" hits eof where NUM was expected.
_EXP_LL1 = {
    "is_ll1": True,
    "start_symbol": "exp",
    "parse_table": {
        "exp": {
            "NUM": [
                {"symbol": "NUM", "field": None},
                {"symbol": "exp2", "field": None},
            ]
        },
        "exp2": {
            "OP": [
                {"symbol": "OP", "field": None},
                {"symbol": "exp", "field": None},
            ],
            "eof": [],
        },
    },
}

    Then add the test:

    def test_incomplete_input_emits_only_one_error_no_cascade(capsys, monkeypatch):
    # Tokens: [NUM("23"), OP("+"), eof] — incomplete, nothing after the operator.
    # The parser fails with found="eof". Without the fix, cursor advances to OP
    # and a second error with found="OP" is emitted (cascade). With the fix,
    # the loop breaks immediately, leaving exactly one error record.
    ll1_file = tempfile.NamedTemporaryFile(mode="w", suffix=".json", delete=False)
    try:
        json.dump(_EXP_LL1, ll1_file)
        ll1_file.flush()
        ll1_file.close()
        tokens = [_tok("NUM", "23"), _tok("OP", "+"), _sentinel()]
        stdin_data = "\n".join(json.dumps(t) for t in tokens) + "\n"
        monkeypatch.setattr("sys.stdin", io.StringIO(stdin_data))
        try:
            run_main([f"--ll1={ll1_file.name}"])
        except SystemExit:
            pass
        out, _ = capsys.readouterr()
        records = [json.loads(l) for l in out.strip().splitlines() if l.strip()]
        err_records = [r for r in records if r.get("kind") == "error"]
        assert len(err_records) == 1
        assert err_records[0].get("found") == "eof"
    finally:
        os.unlink(ll1_file.name)

    • [ ] Step 3: Add the break on found=="eof" in table_cli.py

    In src/plcc/parser/table_cli.py, in the except ParseError block (already modified in Task 1), change cursor += 1 to break when e.found == "eof":

    except ParseError as e:
    record = {
        "kind": "error",
        "message": str(e),
        "stage": "plcc-parser-table",
        "source": e.source,
    }
    if e.found:
        record["found"] = e.found
    print(json.dumps(record), flush=True)
    if e.found == "eof":
        break
    cursor += 1
