# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add automatic dark mode with a manual toggle to the MkDocs Material docs site.

**Architecture:** Add a `palette` block to `mkdocs.yml`. MkDocs Material reads `prefers-color-scheme` media queries to pick the initial theme and persists the user's toggle choice in `localStorage`. No custom code needed.

**Tech Stack:** MkDocs Material (already in use)

## Global Constraints

- Default (no system preference detectable) must be light theme.
- Do not add any custom CSS or JavaScript — use only MkDocs Material config.

---

### Task 1: Add dark mode palette to mkdocs.yml

**Files:**
- Modify: `mkdocs.yml`

**Interfaces:**
- Consumes: nothing
- Produces: a docs site with auto-detecting dark/light theme and a manual toggle

- [ ] **Step 1: Add the palette block**

Open `mkdocs.yml`. The `theme` block currently looks like this:

```yaml
theme:
  name: material
  features:
    - navigation.tabs
    - navigation.sections
    - toc.integrate
    - search.suggest
```

Add a `palette` key inside `theme`:

```yaml
theme:
  name: material
  features:
    - navigation.tabs
    - navigation.sections
    - toc.integrate
    - search.suggest
  palette:
    - media: "(prefers-color-scheme: light)"
      scheme: default
      toggle:
        icon: material/weather-night
        name: Switch to dark mode
    - media: "(prefers-color-scheme: dark)"
      scheme: slate
      toggle:
        icon: material/weather-sunny
        name: Switch to light mode
```

- [ ] **Step 2: Verify the docs site starts cleanly**

```bash
bin/docs/serve.bash
```

Expected: MkDocs starts without errors and prints `Serving on http://127.0.0.1:8000`.
Stop the server with Ctrl-C once it is up.

- [ ] **Step 3: Manual verification**

Run `bin/docs/serve.bash` and open the site in a browser. Check all three scenarios:

1. **System set to dark** — the site loads with the dark (slate) theme automatically.
2. **System set to light** — the site loads with the light (default) theme automatically.
3. **Toggle** — clicking the sun/moon icon in the header switches the theme immediately; reloading the page keeps the toggled choice.

- [ ] **Step 4: Commit**

```bash
git add mkdocs.yml
git commit -m "feat(docs): add dark mode with system-preference detection and toggle"
```
