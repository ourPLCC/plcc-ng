    Design: Issues 017, 023, 029 — Three Small Diagnostic Fixes

    Date: 2026-05-20 Issues: 017, 023, 029 Type: fix, fix, feat

    Overview

    Three independent, self-contained fixes to improve diagnostic output across the pipeline. Each touches a single file at a single logical point.

    Issue 029 — plcc-rep: ^C exits with error instead of silently

    029 Problem

    When ^C is pressed during an interactive plcc-rep session while the REPL is blocked reading a response from the interpreter subprocess, _read_response sees b"" from readline() (the child's stdout closed because SIGINT killed it) and treats this as "interpreter exited unexpectedly", printing an error and exiting with code 1. The desired behaviour is a silent exit with code 130.

    029 Fix

    File: src/plcc/cmd/rep.py, function _read_response

    Before printing the error, call interpreter.poll() to get the exit code. If the code is negative (killed by a Unix signal) or equals 130 (conventional ^C exit), call sys.exit(130) immediately. Otherwise fall through to the existing error path unchanged.

    Add interpreter as a new parameter to _read_response and pass it from RepHandler.feed:

    def _read_response(stdout, interpreter, verbose_format):
    while True:
        raw = stdout.readline()
        if not raw:
            rc = interpreter.poll()
            if rc is not None and (rc < 0 or rc == 130):
                sys.exit(130)
            print('plcc-rep: interpreter exited unexpectedly', file=sys.stderr)
            sys.exit(1)
        ...

    029 Tests

    New test in rep_test.py: simulate the interpreter dying with a negative return code; assert the process exits 130 without printing an error to stderr.

    Issue 017 — plcc-tokens: unrecognized character error omits detail

    017 Problem

    format_error_record in jsonl_formatter.py emits position under the key pos, but print_parse_error in pipeline.py reads record.get("source", {}). The key mismatch silently drops the location. Additionally, the message field does not include the offending character.

    017 Fix

    File: src/plcc/tokens/jsonl_formatter.py, function format_error_record

    1. Rename pos → source.
    2. Include the character in message: f"unrecognized character {char!r}".

    Result:

    {
  "kind": "error",
  "stage": "plcc-tokens",
  "severity": "error",
  "source": {"file": "f.txt", "line": 12, "column": 5},
  "lexeme": "@",
  "message": "unrecognized character '@'"
}

    Rendered by print_parse_error as:

    f.txt:12:5: error: unrecognized character '@'

    017 Tests

    Update existing tests in jsonl_formatter_test.py (or tokens_cli_test.py) to assert source key (not pos) and the updated message string including the character.

    Issue 023 — Tree output does not annotate empty productions

    023 Problem

    When a non-terminal matches an empty production it appears as a leaf node with no children and no visual hint:

    exp
  NUM '1' [-:1:1]
  exp2

    Students may wonder whether exp2 is a mistake or an incomplete parse.

    023 Fix

    File: src/plcc/cmd/parse.py, function _print_tree

    When printing a "tree" node, check whether its children list is empty. If so, append (empty) to the printed rule name.

    if kind == "tree":
    rule = node.get("rule", "?")
    children = node.get("children", [])
    suffix = " (empty)" if not children else ""
    print(f"{prefix}{rule}{suffix}")
    for _field, child in children:
        _print_tree(child, indent + 1)

    Output becomes:

    exp
  NUM '1' [-:1:1]
  exp2 (empty)

    No changes to the JSON tree schema, the parser, or any other pipeline stage.

    023 Tests

    New test in parse_test.py: use a grammar where a non-terminal has an empty production, feed input that triggers it, and assert (empty) appears in stdout for that node but not for nodes with children.

    Summary of changes

    Issue File Change
    029 src/plcc/cmd/rep.py Check interpreter exit code in _read_response before printing error
    017 src/plcc/tokens/jsonl_formatter.py Rename pos → source; include character in message
    023 src/plcc/cmd/parse.py Append (empty) to childless tree nodes in _print_tree
