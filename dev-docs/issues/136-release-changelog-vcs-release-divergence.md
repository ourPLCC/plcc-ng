# 136 - GitHub Release notes and CHANGELOG.md are generated from different sources

**Type:** refactor
**Date:** 2026-07-03

## Description

`pyproject.toml`'s `[tool.semantic_release]` sets `vcs_release: false`, so `python-semantic-release` does not create the GitHub Release itself. Instead, `.github/workflows/release.yml` has a separate step that runs `gh release create --generate-notes`, which builds release notes from merged PRs (GitHub's native auto-notes).

Meanwhile, `CHANGELOG.md` is generated by `python-semantic-release` from conventional-commit messages. These are two independently-generated narratives of the same release and can diverge — e.g. differing grouping, differing commit/PR granularity, or one including something the other omits.

## Notes

- Found while writing the release SOP (issue 130) and reading `release.yml`.
- Possible fix: set `vcs_release: true` and let semantic-release create the GitHub Release using the same changelog content it writes to `CHANGELOG.md`, removing the separate `gh release create` step — unless PR-based release notes are preferred deliberately, in which case this issue can be closed as intentional.
