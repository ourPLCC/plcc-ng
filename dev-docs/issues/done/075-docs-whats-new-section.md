# 075 - Move changelog to a "What's New" section

**Type:** docs
**Date:** 2026-06-07

## Description

The changelog is currently presented as a raw changelog. It should be moved into a "What's New" section that is more user-friendly and discoverable.

## Desired Behavior

- Rename or replace the existing changelog page with a "What's New" section in the docs navigation.
- The content and source of truth for release notes does not need to change — just the presentation and placement.

## Notes

"What's New" is a more user-facing framing than "Changelog," which tends to feel like a developer artifact.
