    Banner Opt-In (--banner / -b) Implementation Plan

    For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

    Goal: Replace the default-on stdout banner with an opt-in --banner / -b flag that prints the version+grammar banner to stderr.

    Architecture: Add print_banner() to output.py, then update each of the five level-2 commands (plcc-make, plcc-scan, plcc-parse, plcc-rep, plcc-diagram) to replace --no-banner with --banner / -b. Orchestrators remove --no-banner from their plcc-make subprocess calls — children are silent by default.

    Tech Stack: Python, pytest (capsys), docopt-ng.

    File Map

    File Change
    src/plcc/cmd/output.py Add print_banner(version, grammar_path, tool, language) → stderr
    src/plcc/cmd/output_test.py Add tests for print_banner
    src/plcc/cmd/make.py --no-banner → --banner / -b; remove pre-docopt stdout print; update banner call
    src/plcc/cmd/make_test.py Delete 4 old banner tests; add 3 new ones
    src/plcc/cmd/scan.py Same pattern as make
    src/plcc/cmd/scan_test.py Delete 4 old banner tests; add 3 new ones
    src/plcc/cmd/parse.py Same pattern
    src/plcc/cmd/parse_test.py Delete 7 old banner tests; add 3 new ones
    src/plcc/cmd/rep.py Same pattern; print_banner also receives tool and language
    src/plcc/cmd/rep_test.py Delete 9 old banner tests; add 4 new ones
    src/plcc/cmd/diagram.py Same pattern; also fix 3 incidental --no-banner usages in unrelated tests
    src/plcc/cmd/diagram_test.py Delete 4 old banner tests; update 3 incidental tests; add 3 new ones
